About Wookie

Wookie enables rapid service development in Scala. Wookie can leverage common technologies such as Akka, Kafka, ZooKeeper, and HTTP services. Wookie can also be used as a library with no container. Wookie is YARN enabled.

Project Modules

This project has declared the following modules:

Name Description
Core Harness Wookie - Core Harness is the base library that allows the container to run or to be embedded in to other applications. The core load the components and services defined through configuration or auto-discovery.
Test Harness Wookie - Test Harness is the base test framework for Wookie that allows easy integration test validation and single node functionality.
Archetype: Service Maven archetype to help jump start services for Wookie.
Archetype: Component Maven archetype to help jump start components for Wookie.
Component: Cache Wookie Cache enables an easy Cacheable trait for in process memory caching. Other cache storage implementations are available such as MemCached.
Component: MemCached Wookie Cache-Memcache enables MemCSached storage for the Cacheable trait.
Component: Cluster Wookie Cluster support enabled by akka clustering and Zookeeper.
Component: Zookeeper Apache Zookeeper support for Wookie, supports leader election and stand Zookeeper node operations.
Component: Kafka Apache Kafka support for Wookie, make simple production and consumption of Kafka topics.
Component: Metrics Standard metric traits to enable easy reporting and data collection on components and services in Wookie.
Component: Netty (HTTP) Apache Netty support for Wookie, enabled the HTTP protocol for Wookie with automatic command routing and simple trait support.
Component: Socko (HTTP) Socko suport for Wookie, supports websockets and standard HTTP protocol with automatic command routing.
Component: Spray (HTTP) HTTP support in Wookie provided by Spray and automatic command routing.
