About Wookiee

Wookiee enables rapid service development in Scala. Wookiee can leverage common technologies such as Akka, Kafka, ZooKeeper, and HTTP services. Wookiee can also be used as a library with no container. Wookiee is YARN enabled.

Project Modules

This project has declared the following modules:

Name Description
Core Harness Wookiee - Core Harness is the base library that allows the container to run or to be embedded in to other applications. The core load the components and services defined through configuration or auto-discovery.
Test Harness Wookiee - Test Harness is the base test framework for Wookiee that allows easy integration test validation and single node functionality.
Archetype: Service Maven archetype to help jump start services for Wookiee.
Archetype: Component Maven archetype to help jump start components for Wookiee.
Component: Cache Wookiee Cache enables an easy Cacheable trait for in process memory caching. Other cache storage implementations are available such as MemCached.
Component: MemCached Wookiee Cache-Memcache enables MemCached storage for the Cacheable trait.
Component: Cluster Wookiee Cluster support enabled by akka clustering and Zookeeper.
Component: Zookeeper Apache Zookeeper support for Wookiee, supports leader election and stand Zookeeper node operations.
Component: Kafka Apache Kafka support for Wookiee, make simple production and consumption of Kafka topics.
Component: Metrics Standard metric traits to enable easy reporting and data collection on components and services in Wookiee.
Component: Netty (HTTP) Apache Netty support for Wookiee, enabled the HTTP protocol for Wookie with automatic command routing and simple trait support.
Component: Socko (HTTP) Socko suport for Wookiee, supports websockets and standard HTTP protocol with automatic command routing.
Component: Spray (HTTP) HTTP support in Wookiee provided by Spray and automatic command routing.
Component: Etcd Etcd component allows discovery of Wookiee Services running in docker using etcd.
